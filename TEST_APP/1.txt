## üìÇ TASKS TO EXECUTE

### 1) FULL PROJECT SCAN
Read the entire repository structure.

Produce:
- high level tree
- important folders
- suspicious folders
- dead-looking zones
- duplicated areas

---

### 2) DETECT TECHNOLOGY STACK
You must identify:

Backend:
- language
- framework
- ORM
- auth system
- API style

Frontend:
- framework
- routing
- state management
- API layer

Other:
- cache
- queue
- cron
- file storage
- external APIs

---

### 3) FIND ENTRY POINTS
Where does execution start?

Examples:
- server bootstrap
- app.ts
- main.py
- index.js
- router root
- CLI start

---

### 4) CONFIGURATION & ENVIRONMENT
Find:
- env files
- secrets risks
- hardcoded credentials
- environment switching logic

---

### 5) IDENTIFY ARCHITECTURE STYLE
Try to understand:

Is it:
- layered?
- feature based?
- monolith?
- modular?
- chaotic?

Explain WHY.

---

### 6) LOCATE BUSINESS LOGIC
Where is REAL value implemented?

Not routing.
Not UI.

Where decisions happen.

---

### 7) RISK HEATMAP
Mark areas likely to explode in future:

Examples:
- huge files
- mixed responsibilities
- deep nesting
- unclear naming
- magic constants
- global states

---

### 8) DEPENDENCY DIRECTION
Check:

Does UI depend on domain?
Does domain depend on infra?
Are there circular imports?

---

### 9) ONBOARDING SIMULATION
Imagine a new developer comes tomorrow.

Explain:
- where he will be confused
- what is not obvious
- what is missing
