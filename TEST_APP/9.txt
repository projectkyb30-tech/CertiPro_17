## üìÇ TASKS TO EXECUTE

---

### 1) LOGGING STRATEGY
Check:
- where logs are written
- levels (info/warn/error/debug)
- contextual data (user id, request id)
- structured vs random logs
- correlation between services
- important events logged or not

If errors happen but no logs ‚Üí critical operational risk.

---

### 2) ERROR VISIBILITY
Check:
- global error handling
- uncaught exception handling
- promise rejection handling
- frontend crash capture
- silent failures

If failures can disappear ‚Üí High risk.

---

### 3) AUDIT TRAIL FOR IMPORTANT ACTIONS
For business critical operations check:

Examples:
- payments
- order submission
- exam submission
- account changes
- permission changes

Is it logged who did what and when?

If not ‚Üí forensic nightmare.

---

### 4) METRICS & MEASUREMENT
Look for signals like:
- request durations
- error rate
- throughput
- queue size
- retries
- failed jobs

If no metrics ‚Üí cannot manage growth.

---

### 5) HEALTH CHECKS
Check existence of:
- health endpoints
- readiness/liveness
- DB connectivity checks
- dependency checks

Without this, orchestration and uptime suffer.

---

### 6) ALERTING POSSIBILITY (IF CONFIGS EXIST)
If project hints at monitoring:
- what would trigger alerts?
- would anyone know in minutes?

If outage lasts hours unnoticed ‚Üí major risk.

---

### 7) DEBUGGABILITY
Imagine production bug.

Can engineer quickly:
- find request
- find user
- replay scenario
- trace operations

If not ‚Üí slow incident recovery.

---

### 8) DATA FOR SUPPORT TEAM
Check:
- can support see status?
- can they explain failure to user?
- or they guess blindly?

---

### 9) BACKGROUND JOB VISIBILITY (if present)
If queues/cron:
- logging
- retries
- dead letter handling
- monitoring

Silent background failure = huge hidden risk.

---

### 10) CONFIGURATION VISIBILITY
Can team know:
- what version runs?
- what config active?
- what env variables loaded?

---

## üß† ADVANCED DETECTIONS

Detect:
- logs without identifiers
- inconsistent log formats
- catching errors but ignoring them
- retry loops without tracking
- operations with no traceability
- inability to link frontend issue to backend request
