## üìÇ TASKS TO EXECUTE

---

### 1) IDENTIFY ACTORS
Who interacts with the system?

Examples:
- guest
- user
- admin
- operator
- system
- external service

For each actor:
- what can they do?
- where?
- how?

---

### 2) MAIN BUSINESS FLOWS
Find top flows.

Examples:
- registration
- purchase
- booking
- order processing
- exam start / finish
- payment
- approval

For each flow describe:

- start point
- steps
- validations
- state changes
- final result

---

### 3) DOMAIN ENTITIES
Find core models.

Examples:
- user
- order
- exam
- booking
- product

For each entity:
- what defines it?
- what changes it?
- lifecycle
- who can modify it?

---

### 4) BUSINESS RULES
Find hidden laws.

Examples:
- cannot pay twice
- must be logged in
- must pass validation
- cannot edit after submit
- stock must exist

If rule is not enforced ‚Üí CRITICAL.

---

### 5) DECISION POINTS
Where system makes choices.

Examples:
- price calculation
- access control
- result computation
- availability
- status transitions

Mark complexity level.

---

### 6) VALIDATION LOGIC
Where and how inputs are verified.

Missing validation = danger.

---

### 7) STATE MACHINE
Understand transitions.

Example:
draft ‚Üí submitted ‚Üí approved ‚Üí archived

If unclear ‚Üí big risk.

---

### 8) EDGE CASE DETECTION
Think like attacker or idiot user.

What if:
- double click?
- network drop?
- refresh?
- invalid input?
- race condition?

---

### 9) BUSINESS DEPENDENCY GRAPH
Who depends on what.

If changing one thing breaks 20 ‚Üí problem.

---

### 10) SIMPLICITY TEST
Could logic be simpler?

If yes ‚Üí mark improvement.

---
